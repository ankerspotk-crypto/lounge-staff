## 0. 要約

- 土台は **Googleスプレッドシート2冊 ＋ タブ約40枚**。構造そのものはシンプル。✅
- 問題は**中央の窓口が無いこと**。`SpreadsheetApp.openById(SHEET_ID)` が **100回超インラインで直叩き**。ヘルパー `getOrOpenSS_()`（`Code.gs:7511`付近）は**存在するのに数箇所しか使われていない**＝各関数が勝手に取りに行く。参照先の変更が全所へ飛び火するのが「都度指摘」の主因。🔶
- 全システムを貫く**照合キーは2本だけ**：**会員番号**（顧客⇄売上）と **源氏名**（スタッフ⇄シフト/給与/勤怠）。指摘はこの結合点に集中。✅
- **散らかりの震源は7つ**（→ 第4章）。優先度＝高3・中3・低1。
- 片付けは**段階式・非破壊**で、全て**テスト環境ファースト**。本番は「本番に移して」の号令が出た時だけ（`feedback_test_env_first`）。

## 3. 全体をつなぐ照合キー（背骨・✅確定）

### 会員番号 — 顧客 ⇄ 売上
- **お客様管理Y3** を **来店記録・TRUST報酬・売上明細・伝票** に結ぶ。
- **正は「番号のみ」**（4桁運用・照合は先頭ゼロ無視）。
- **地雷**：名字フォールバック照合が同名別人の売上を混線させた（@685）→ **番号のみ**に修理済（@686）。誤りは読取り時だけで保存は無傷。仕訳タブで付替え可（`reference_visit_attribution_matching`）。

### 源氏名 — スタッフ ⇄ シフト／給与／勤怠
- **スタッフマスタ** を **シフト表・給与計算・勤怠ログ・誕生日バック** に結ぶ。派遣は**派遣名マッピング**経由。
- **地雷**：`normalizeName_` が内部スペースを消さない＝**「鈴木 海」と「鈴木海」が別人**化。照合系は空白除去キーを自前で作る（`reference_name_normalization`）。

## 4. 散らかりの震源（参照バラバラの正体・優先度つき）

| 優先 | 震源 | 中身 |
|---|---|---|
| **高** | **中央アクセサ不在** | `openById(SHEET_ID)` が100回超インライン。`getOrOpenSS_()` は在るのに不採用。参照先変更が全所へ飛び火。✅ |
| **高** | **席＝3台帳合成＋日付定義2種** | 予約管理／RSRV_客名コピー／在席 を毎回合成。客名=営業日・在席=暦日で0〜6時に1日ズレ。「ご来店」誤表示＝台帳破損サイン（`reference_seat_state_ledgers`）。 |
| **高** | **売上が2パイプライン非交差** | 来店記録(per-visit) と TRUST取込(月次×給与) が別系統でUI導線が片方に無く、軍師来店の売上が永久に空（`reference_visit_sales_two_pipelines`）。 |
| 中 | シフト2ブック分裂 | 表=別冊／申請=本体。跨ぐ関数あり。 |
| 中 | 照合キーの不統一 | 会員番号(番号のみ)・源氏名(空白/派遣名)で個別実装が散る。共通ヘルパー未徹底。 |
| 中 | 退職者フィルタ複数経路 | 在籍者を引く経路が画面用と通知用で別実装。除外もれ（`reference_shift_retire_filter`）。 |
| 低 | 旧タブの化石同居 | 在庫発注マスタ_旧／面談カルテ(旧名) 等。命名も `_TAB` 定数と直書き文字列が混在。 |

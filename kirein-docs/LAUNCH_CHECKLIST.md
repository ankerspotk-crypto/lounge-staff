## 🚀 本番前に必須の作業

### 🔴 CRITICAL（開始前に必ず対応）

1. **Firestore セキュリティルール検証**
   - テスト店舗で、ネガティブ情報が正しく制限されているか確認
   - 投稿者以外の消費者がネガティブ情報を見られないか確認

2. **Stripe webhook 設定**
   - 決済完了時に `stores/{uid}` の `is_paid: true` に自動更新
   - テスト決済で動作確認

3. **エラーハンドリング強化**
   - Firestore 接続エラー時のユーザーメッセージ
   - ネットワーク遅延時のローディング表示

4. **モバイル対応確認**
   - iPhone Safari / Android Chrome で投稿フロー・ダッシュボードが正常に表示されるか

### 🟡 HIGH（開始から1週間以内）

5. **ログ・モニタリング設定**
   - Firebase Cloud Logging を有効化
   - エラー発生時に通知を受ける仕組み

6. **データバックアップ手順**
   - Firebase の自動バックアップ確認
   - 手動バックアップの実施方法を文書化

7. **ユーザーサポート連絡先**
   - メールアドレス / LINE / 問い合わせフォームを決定
   - owner.html / index.html に記載

8. **GDPR / 個人情報保護法確認**
   - 弁護士レビュー（推奨）
   - データ保持期限・削除ポリシーを決定

### 🟢 MEDIUM（開始から1ヶ月以内）

9. **複数ユーザーでのテスト**
   - 複数の消費者 / 複数の店舗オーナーでの実運用テスト

10. **メール通知の実際の動作**
    - 実ユーザー投稿後、メール通知が正しく送られるか確認
    - メール送信上限（1日100通）を監視する仕組み

11. **分析・メトリクス**
    - GA4 設定（ユーザー行動追跡）
    - 投稿数 / ユーザー数 / NPS の計測

AWS セットアップガイド

キレイン向けの最小構成。結論：「AWS フリープラン」で十分。

🎯 結論

「Free Tier（フリープラン）」を選んでください。

AWS 新規アカウント＆クレジットカード登録で、12ヶ月間は大部分のサービスが無料。
キレインの初期段階（Month 1-3）は、ほぼ料金 ¥0 で運用可能。

1. 契約フロー

Step 1：AWS アカウント開設（5分）

  1. AWS 公式サイト（aws.amazon.com/jp）にアクセス
  2. 「アカウント作成」をクリック
  3. メールアドレス + パスワード登録
  4. クレジットカード登録（審査なし、即座に完了）
  5. 電話番号確認（SMS）
  6. 完了：フリープラン自動適用

Step 2：サポートプラン選択

プラン料金推奨
Basic（無料）¥0✓ 初期段階向け
Developer月 ¥29-
Business月 ¥500+-
Enterprise月 ¥15,000+-
→ 「Basic（無料）」を選択してください。
Month 1-6 は Basic で十分。エラー時は AWS Documentation や Stack Overflow で十分解決できます。

4. アカウント開設後のセットアップ（チェックリスト）

4.1 セキュリティ設定（必須）

  • ☐ 2 要素認証（2FA）を有効化
  • ☐ IAM ユーザーを作成（ルートアカウントは使わない）
  • ☐ IAM ユーザーに以下の権限を付与：
    • Lambda：完全アクセス
    • DynamoDB：完全アクセス
    • CloudWatch：完全アクセス
    • IAM：読み取りのみ
  • ☐ AWS Budgets で「月 ¥1,000」のアラート設定（超過防止）

4.2 サービス設定

  1. Lambda 関数の作成
    • 関数名：kirein-post-trigger
    • ランタイム：Node.js 18.x（または Python 3.11）
    • トリガー：API Gateway or DynamoDB Stream
  2. DynamoDB テーブルの作成
    • テーブル 1：users（ユーザー情報）
    • テーブル 2：posts（投稿データ）
    • テーブル 3：stores（店舗情報）
    • 課金モード：オンデマンド（自動スケーリング）
  3. CloudWatch ダッシュボード
    • Lambda エラー率
    • DynamoDB の読み取り / 書き込み数
    • API レスポンスタイム

5. 代替案：「AWS 不要」の選択肢

⚠️ 実は「AWS なしで始める」という選択肢もあります

代替案：Firebase（Google）を使う

✓ セットアップが簡単（5分で完了）
✓ 無料枠が充実（年間¥0～¥10,000まで無料）
✓ データベース + 認証 + ホスティング一体型
✗ カスタマイズ性が劣る

Firebase を選ぶ場合：
Firestore（NoSQL DB）+ Cloud Functions（サーバーレス） + Authentication

8. 無料枠をオーバーする場合の対策

シナリオ対策
Lambda 呼び出しが 200万超過DynamoDB Stream で効率化（非同期処理）
DynamoDB が 25GB を超えるS3 へのアーカイブ or テーブル分割
CloudWatch ログが 5GB 超ログ保持期間を 7 日に短縮
実際には Month 1-6 で超過する可能性は低い

投稿データは 1 件 = 1KB程度なので、20,000 件投稿しても 20GB にしかならない（無料枠内）。

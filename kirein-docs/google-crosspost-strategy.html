Google Maps クロスポスト戦略

既存の Google レビュアーを Kirein へ自然流入させる仕組み

1. 戦略概要

コンセプト
「Google Maps で既に評価している人」に対して、Kirein への投稿を促す。

メリット
✓ Google レビュアー = 既に "きれいさ" に関心のあるユーザー層
✓ 一度のレビュー作業で両方に投稿 = 摩擦が少ない
✓ Google のデータ = 信頼性が高い
✓ 友達紹介より CAC が低い可能性

2. ユーザーフロー

Scenario A: Google → Kirein（推奨）
1. ユーザーが Google Maps で飲食店のレビュー投稿を完了
2. 投稿直後に CTA：「Kirein でもこのお店を評価してトイレの清潔さを記録する」
3. Google のレビュー内容を一部コピーして Kirein に投稿
4. ★評価（トイレ・空気）を入力して完了
5. ポイント 150pt 獲得（キャンペーン中）
Scenario B: Kirein の投稿から Google への誘導
1. ユーザーが Kirein で投稿完了
2. 完了画面に「Google Maps でもこのお店をレビューして評判を広める」CTA
3. Google Maps の店舗ページへのリンク
4. Google でのレビュー投稿数 ↑ → Kirein のデータ信頼性向上

3. 実装方法

3.1 Google Maps API との連携

実装項目内容難易度
店舗自動同期Kirein に掲載されている店舗が Google Maps に存在するか自動確認中
Google レビュー取得Place Details API で Google レビュー平均点を Kirein に表示中
クロスポスト CTA投稿完了後に「Google でもレビュー」ボタンを表示易
デバイス間連携ユーザーが投稿した内容を一時保存 → Google モバイルアプリへ自動遷移中

3.2 技術スタック

  • Google Maps Platform - Places API（店舗検索・レビュー取得）
  • Google Search Console API - Kirein サイトが Google で どう見えているか
  • Universal Links / App Links - iOS/Android デバイスで Google Maps アプリへシームレス遷移
  • Open Graph Tags - Kirein 投稿を Google+ / SNS にシェア可能に
注意：Google の規約
- Kirein → Google への自動投稿は NG（Google が自動投稿を禁止）
- ユーザーが手動で Google にリンク遷移 → 投稿する のは OK
- Google レビュー内容の引用・コピペ表示は OK（ただし著作権に配慮）

4. ユーザー体験

4.1 Google から Kirein へのクロスポスト（推奨フロー）

ステップ画面/テキスト
1. Google 投稿完了ユーザーが Google Maps で「鮨 清水 ★★★★★」と評価投稿
2. ポップアップ表示「Kirein でもこのお店を評価しませんか？ 150pt ボーナス！」
3. リンククリック→ Kirein アプリ/サイトに自動遷移、店舗は「鮨 清水」が自動選択済み
4. 評価入力Google での総合評価 ★★★★★ を参考に、「トイレ清潔 ★★★★☆」「空気・無臭 ★★★★★」を入力
5. コメント入力Google での「カウンターも手洗いも気持ちいいくらい清潔。接待に最適です。」をそのまま引用
6. 投稿完了「+150pt 獲得！」 & Google でのレビューが Kirein にも記録される

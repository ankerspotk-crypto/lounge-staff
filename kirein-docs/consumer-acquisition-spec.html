消費者獲得システム仕様書 v1.0

目標：初月5,000件の投稿獲得 → レポーター・調査員パイプライン構築

1. ポイント・Lvシステム

1.1 ポイント付与ルール

シーン通常キャンペーン中その他
投稿1件50pt150pt（3倍） or 250pt（5倍）—
高評価（5★）+10pt+30pt（3倍）実績ベース
位置情報付き+5pt+15pt—
写真付き+10pt+30pt将来検討

1.2 Lvシステム（レポーター昇格）

Lv必要ポイント投稿数（通常）投稿数（キャンペーン中）ステータス
00pt——ビジター
1500pt10件2〜3件認定レポーター
22,000pt40件8〜10件見習い調査員（選抜）
35,000pt100件20〜25件公認覆面調査員
自動判定ロジック
ユーザーのポイント が Lv.N の必要ポイント に到達 → 自動で Lv.N に昇格 → 自動DM送信「おめでとうございます！」

1.3 Lv.2（見習い調査員）への昇格条件

  • 必須：Lv.1で 20件以上の投稿実績
  • 必須：整合度スコア 90% 以上（プロの評価と一致度）
  • 条件：本人確認（電話 or 身分証）
  • 条件：利用規約同意
  • 判定：キレイン側で月1回、手動選抜

2. 調査員報酬・業務設計

2.1 見習い調査員（Lv.2）

項目内容
報告書報酬1件あたり ¥1,000（指定フォーマットで報告）
月の期待件数2〜3件（無理強いしない）
月の獲得額目安¥2,000-3,000
報告項目トイレ清潔度・空気・接客態度・全体評価・写真
契約業務委託契約（試用3ヶ月）

2.2 公認調査員（Lv.3）

項目内容
報告書報酬1件あたり ¥3,000（指定フォーマットで報告）
月の期待件数4〜6件
月の獲得額目安¥12,000-18,000
報告項目詳細レポート（トイレ・空気・接客・衛生管理）+ 改善提案
特典専用ダッシュボード、優先案件配分
契約本契約（年1回更新）
見習い→本契約への昇格
試用3ヶ月で「実績・精度・対応」が基準をクリア → Lv.3へ昇格提案

3. 初月5,000件投稿の達成シミュレーション

3.1 投稿の内訳想定

  • TikTok施策：フォロワー数 10万達成時点で月 2,000〜3,000件見込み
  • Instagram施策：月 1,500〜2,000件見込み
  • 口コミサイト（食べログ等）：クロスポスト誘導で 500〜1,000件
  • 既存加盟店からの投稿：月 200〜300件

3.2 キャンペーン戦略

Phase 1（Week 1-2）：告知・認知
TikTok / Instagram で「無料で調査員になれる」キャンペーン開始
インフルエンサー、飲食店アカウントでシェア

Phase 2（Week 3-4）：投稿誘導
「今だけ投稿ポイント3倍〜5倍」の限定感で急かす
ポイント到達者に「レポーター認定おめでとう」DM

Phase 3（Month 2-）：継続投稿
Lv.1→Lv.2へのアップセルDM
「見習い調査員募集中、月¥5,000」で有料化へ導く

3.3 5,000件到達時点でのデータ

  • 投稿ユーザー数：3,000〜4,000人（重複除く）
  • Lv.1到達者：500〜1,000人
  • 見習い調査員候補：50〜100人
  • 掲載店舗：1,000〜1,500軒（全国）
  • Lv分布：Lv.1 = 投稿の10〜15%、Lv.2+ = 1〜2%

4. DBスキーマ・実装詳細

4.1 ユーザーテーブル

users
├ user_id (PK)
├ phone_or_line_id
├ nickname
├ profile_image
├ total_points
├ current_lv (0-3)
├ lv_update_date
├ integrity_score (整合度)
├ post_count
├ verified (本人確認済み)
├ created_at
└ updated_at

4.2 ポイント・Lvロジック

【自動判定】
WHEN user.total_points >= Lv.N_threshold
  THEN user.current_lv = N
  THEN auto_send_dm("Lv.N認定！")
  THEN trigger_lv_milestone()

4.3 見習い調査員の昇格フロー

【月1回のバッチ処理】
SELECT * FROM users
WHERE current_lv = 1
AND post_count >= 20
AND integrity_score >= 0.90
AND verified = true
THEN suggest_apprentice_recruit()
THEN send_dm_with_contract()
